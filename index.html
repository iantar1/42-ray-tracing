MiniRT

A physically-based ray tracer built in C++ with the MiniLibX display library. Renders spheres, cylinders, cones, and planes with Phong lighting, shadows, and recursive reflections.

01 — Overview

What is RT?

RT is a CPU ray tracer written in C++. It simulates how light travels and bounces in a 3D scene to produce photorealistic images. For every pixel on screen, a ray is shot from the camera into the scene; the renderer finds what it hits, computes diffuse and specular lighting, casts shadow rays to each light, and — if the surface is reflective — recursively follows the bounced ray.

03 — Core Math & Physics

Vectors, Rays, and Geometry

3D Vectors — Vec3

The Vec3 class is the numeric backbone of the renderer. Every position, direction, and color is a Vec3. Points3 is a type alias for it — the distinction is purely semantic (a point vs. a direction).

// Dot product — measures how aligned two vectors are v₁ · v₂ = x₁x₂ + y₁y₂ + z₁z₂ // Returns a scalar. Positive ⟹ same direction, 0 ⟹ perpendicular, negative ⟹ opposite.
// Cross product — vector perpendicular to both inputs v₁ × v₂ = ( y₁z₂−z₁y₂, z₁x₂−x₁z₂, x₁y₂−y₁x₂ )
// Normalization — make length = 1 (unit vector) v̂ = v / |v| where |v| = √(v · v)

The Ray — Ray.hpp

A ray is a parametric half-line starting at an origin O heading in direction D:

P(t) = O + t · D (t ≥ 0) // O = ray origin (camera position or reflection point) // D = ray direction (unit or non-unit; checked per-intersector) // t = distance parameter — larger t ⟹ farther along the ray // Ray::at(t) computes P(t) and returns it as a Points3
Ray Parameterisation

Rodrigues' Rotation Formula

Used in utils.hpp → rotateAroundAxis() and by the light movement system to rotate a vector around an arbitrary axis without building a full rotation matrix:

// Rotate vector v by angle θ around unit axis k: v′ = v cosθ + (k × v) sinθ + k (k · v)(1 − cosθ) // Term 1: the component of v in the plane, scaled by cosθ // Term 2: perpendicular rotation component (cross product) // Term 3: the axial component k(k·v) unchanged times (1−cosθ) correction
Rodrigues' Rotation — decomposition of v into axial + perpendicular components
04 — Geometric Objects

Ray–Object Intersection

Each shape implements intersect(const Ray& ray, double& t_hit). The function solves the algebraic equation P(t) ∈ surface for the smallest positive t.

🔵 Sphere

A sphere is the set of points at distance r from center C:

// Sphere implicit equation: |P − C|² = r² // Substitute P = O + tD (let oc = O − C): |oc + tD|² = r² (D·D)t² + 2(oc·D)t + (oc·oc − r²) = 0
// Quadratic: at² + bt + c = 0 a = D·D, b = 2(oc·D), c = oc·oc − r² Δ = b² − 4ac // Δ < 0 → miss. Δ ≥ 0 → t = (−b ± √Δ) / (2a) // Pick smallest positive t. Normal = normalize(P − C).

🟦 Plane

A plane is defined by a point p₀ on it and a normal n. Every point P in the plane satisfies (P − p₀) · n = 0.

// Substitute the ray: (O + tD − p₀) · n = 0 t = ( (p₀ − O) · n ) / (D · n) // If |D·n| < ε → ray is parallel to plane → no intersection. // Normal is constant: normalize(n). t ≥ 0 required.

🟧 Cylinder

An infinite cylinder with center C, unit axis â, and radius r. The trick is to decompose the ray and offset vector into components perpendicular to the axis — the axial component doesn't affect the radius.

// Project D and oc = O−C onto the plane ⊥ to axis: D⊥ = D − â (D·â) oc⊥ = oc − â (oc·â) // Solve 2D circle intersection in that plane: a = D⊥·D⊥, b = 2(oc⊥·D⊥), c = oc⊥·oc⊥ − r² Δ = b² − 4ac // After finding t, clamp to finite height: |( P(t) − C ) · â| ≤ half_height // Normal = normalize( oc − â(oc·â) ) — radial vector from axis to surface.

🔺 Cone

A cone with apex A, unit axis â, and half-angle θ. A point P is on the surface when the angle between (P−A) and â equals θ.

// Cone surface condition (squared to remove absolute value): ((P−A)·â)² = cos²(θ) · |P−A|² // Substitute P = O + tD, oc = O − A: a = (D·â)² − cos²θ (D·D) b = 2[ (D·â)(oc·â) − cos²θ (oc·D) ] c = (oc·â)² − cos²θ (oc·oc) // Solve at² + bt + c = 0, pick smallest positive t. // Clamp: projection = (P−A)·â must be in [0, half_height]. // Also intersect the circular base disk at projection = half_height. // Normal = normalize(perpendicular component of P−A from axis).
Cross-section: Ray hitting a Sphere, Cylinder, Cone, and Plane
05 — Lighting

Phong Illumination Model

The Phong model splits the light arriving at a surface point into three components: ambient (constant base brightness so nothing is fully black), diffuse (matte, angle-dependent), and specular (shiny highlight).

// For each light L with intensity Iₗ: I = Iₐ·Cₒ + Σ Iₗ [ Kd·Cₒ·(N̂·L̂) + Ks·(R̂·V̂)ⁿ ]
// Symbols: // Iₐ = ambient intensity (0.2 × object color when in shadow) // Cₒ = object color (RGB 0–255) // N̂ = surface normal (unit vector, outward) // L̂ = direction to light source (normalized) // R̂ = reflection of L̂ about N̂ = 2(N̂·L̂)N̂ − L̂ // V̂ = view direction (from hit point to camera, normalized) // n = shininess exponent (higher → sharper highlight) // max(0,…) clamps negative dot products (light behind surface)
Phong Shading Geometry at a Surface Point

Lambert Diffuse

Brightness is proportional to the cosine of the angle between the surface normal N̂ and the light direction L̂. This is N̂ · L̂. A surface facing the light directly (θ = 0°) gets full brightness; grazing angles give near zero.

Phong Specular

The specular term depends on how close the viewer V̂ is to the perfect mirror direction R̂. The exponent n (shininess) controls the tightness. High shininess (e.g. 64) = small sharp metallic highlight. Low shininess (e.g. 4) = large diffuse glow.

Multiple Lights & Attenuation

The scene accumulates contributions from every light. Diffuse and specular totals are clamped to [0, 1] to prevent oversaturation, then combined into the final RGB pixel value:

int r = (int)(diffuse * color.r + specular * 255.0);
int g = (int)(diffuse * color.g + specular * 255.0);
int b = (int)(diffuse * color.b + specular * 255.0);

Specular uses white (255) because highlights on most surfaces appear white regardless of the object's base color.

06 — Shadows & Reflections

Shadow Rays and Recursive Reflections

Shadow Testing

After finding a hit, a shadow ray is cast from the hit point toward each light. If any other object blocks the path, the contribution of that light is replaced by ambient-only lighting.

// Shadow ray: S(t) = P_hit + ε·L̂ + t · L̂ // ε (epsilon = 1e-4) offsets origin slightly to avoid self-intersection. // Only count an occluder if: ε < t < distance_to_light // (prevents objects behind the light from casting shadows)

Reflections — Recursive Ray Casting

If an object has reflectivity > 0, a reflected ray is spawned. The color contributed by the reflection is blended with direct lighting using linear interpolation:

// Reflected ray direction (incoming ray I hits surface normal N): R = I − 2(I · N̂) N̂ // (The component of I along N is reversed; the tangential component unchanged)
// Color blending: C_final = C_direct · (1 − reflectivity) + C_reflected · reflectivity // Recursion terminates at depth = 3 (returns background color)
Recursive Reflection (depth = 2 shown)
Depth limit: Recursion is capped at depth 3. At depth ≥ 3 the function returns the background color (blue). This prevents infinite loops when two mirrors face each other and keeps render times bounded.
07 — Camera

Perspective Camera & Ray Generation

The camera uses a simple pinhole / perspective model. Each pixel (x, y) is mapped to a normalized device coordinate (NDC) in [−1, 1]², then a ray is fired through that screen point.

// NDC mapping — centre of pixel (x,y) in [IMG_WIDTH × IMG_HEIGHT]: px = (2x / W) − 1 maps [0, W] → [−1, +1] py = 1 − (2y / H) maps [0, H] → [+1, −1] (Y flipped) // Ray direction (in camera space, z forward = −1): D = (px, py, −1) (not normalized — intersectors handle that) // The ray origin is the camera position.
Camera → Image Plane → Scene (perspective projection)

Camera Movement

Three movement functions let you navigate the scene at runtime. They all rebuild the camera's position vector using the current orientation:

MethodMathDescription
moveForward(d)pos += normalize(dir) · dMove along look direction
moveRight(d)pos += normalize(dir × (0,1,0)) · dStrafe using cross product with world-up
moveUp(d)pos += (0,1,0) · dMove along world Y axis

Object selection works by firing a pick ray through the clicked pixel (same getRay(x, y) as rendering), finding the closest intersection, and storing a pointer to that Objects*. Transform keys then operate on that stored pointer via the polymorphic translate() / rotateX/Y/Z() interface.

Scene Configuration (main.cpp)

Objects and lights are added programmatically in main():

// Sphere: radius, center, color, shininess, transparency, reflectivity
scene.addObject(make_unique<Sphere>(1.0, Points3(2,1,-5), Vec3(255,0,0), 64.0, 0.0, 0.6));

// Cone: apex, axis, half_angle, half_height, color, shininess, transparency
scene.addObject(make_unique<Cone>(Points3(0,-1,-2), Vec3(0,1,0), 0.4, 2.0, Vec3(0,255,0), 16.0, 0.5));

// Plane: normal, point, color, shininess, transparency, reflectivity
scene.addObject(make_unique<Plane>(Vec3(0,1,0), Points3(3,-1,-6), Vec3(255,255,0), 32.0, 0.0, 0.7));

// Light: intensity, angle (unused), position
scene.addLight(Light(1.0, 0, Vec3(-1,1,-1)));
scene.addLight(Light(0.8, 0, Vec3(2,2,-2)));
